Constantes em javascript

Criamos uma constante, declarando um variavel atraves da palavra const.
As constantes são valores imutaveis durante todo o ciclo da aplicação.
Assim como em let, a constante tambem atua no escopo de bloco. Sendo assim, ela existe dentro do escopo do bloco em que foi definida.

Constantes em um bloco

Aqui, temos uma constante definida dentro de um bloco.
if(true){
const nome = "Andre";
console.log(nome);
}

Se tentarmos invocar esta constante fora do bloco em que foi criado, teremos um erro de interpretação.
if(true){
const nome = "Andre";
console.log(nome);
}
console.log("Constante dentro de bloco, sendo chamada fora dele");
console.log(nome);

Redeclaração e Reatribuição

Redeclarar é recriar uma variavel com um mesmo nome, dentro do mesmo escopo.

Reatribuição é dar um novo valor a uma variavel ja criada

Quando criamos uma variavel com var, podemos redeclarar e redefinir seu valor.
var xpto = 'xpto';
var xpto = 'AAA';

Quando criamos uma variavel com let, podemos redefinir, porem não redeclarar.
let xpto = 'xpto';
xpto = 'AAA';

Ja quando criamos uma variavel com const, não podemos nem redeclarar nem redefinir, isso dentro do mesmo escopo.
const xpto = 'xpto';
const xpto = 'AAA';

Porem, em escopos diferentes, essa redeclaração é possivel.
if(true){
const xpto = 'xpto';
}
const xpto = 'AAA';

Existe uma particularidade dentro do javascript, que quando voce cria uma constante, como um objeto ou um array, voce pode modificar suas propriedades e valores, desde que, a constante em não seja redeclarada.
const dominios = ['JS', 'Python', 'Go'];
> (3) ["JS", "Python", "Go"]

dominios[1] = 'C#';
>(3) ["JS", "C#", "Go"]
